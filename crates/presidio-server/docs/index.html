presidio-rust — de-identification API

Self-hosted, ISO-compatible with the Limina / Private AI v4 API. Stateless, container-friendly, CPU-able. Point your existing PrivateAI/Limina client at this server — same request/response shape.

Base URL: http://<host>:8080 · Content-Type: application/json · No data leaves your infrastructure.

POST /process/text ISO

Request

{
  "text": ["Email John at john@acme.com or call 415-555-0198"],
  "link_batch": false,
  "entity_detection": {
    "accuracy": "high",
    "entity_types": [{ "type": "ENABLE", "value": ["EMAIL_ADDRESS", "PHONE_NUMBER"] }]
  },
  "processed_text": { "type": "MARKER", "mask_character": "#" }
}
FieldNotes
textarray of strings to process (batch)
entity_detection.accuracyhigh / standard (accepted; engine is single-config)
entity_detection.entity_types[{type:"ENABLE"|"DISABLE", value:[...]}] — restrict labels
processed_text.typeMARKER → [LABEL_N], MASK → repeated mask_character, SYNTHETIC → <LABEL>

Response (array, one item per input text)

[
  {
    "entities": [
      { "processed_text": "[EMAIL_ADDRESS_1]", "text": "john@acme.com",
        "best_label": "EMAIL_ADDRESS", "labels": { "EMAIL_ADDRESS": 0.85 },
        "location": { "stt_idx": 14, "end_idx": 27, "stt_idx_processed": 14, "end_idx_processed": 31 } }
    ],
    "entities_present": true,
    "characters_processed": 48,
    "languages_detected": { "en": 1.0 },
    "processed_text": "Email John at [EMAIL_ADDRESS_1] or call [PHONE_NUMBER_1]"
  }
]

location.stt_idx/end_idx = character offsets in the source; *_processed = offsets of the marker in processed_text.

curl

curl -s -X POST http://localhost:8080/process/text \
  -H 'content-type: application/json' \
  -d '{"text":["call 415-555-0198"],"processed_text":{"type":"MARKER"}}'

Migrating from Private AI / Limina

Change only the base URL. Request/response fields (text, entity_detection, processed_text, entities[].best_label, location.stt_idx/end_idx, processed_text) are identical. Benchmarked ≥ PrivateAI quality on ai4privacy / WikiANN / verified-multilingual, far stronger on non-Latin scripts, at self-hosted cost.

Entity labels

Canonical labels: NAME_GIVEN, NAME_FAMILY, LOCATION_CITY, LOCATION_COUNTRY, ORGANIZATION, EMAIL_ADDRESS, PHONE_NUMBER, DATE, IP_ADDRESS, CREDIT_CARD, SSN, IBAN, URL, … Full list at GET /supportedentities.
